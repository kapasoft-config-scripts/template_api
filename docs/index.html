Template API

Issue & Solution

Issue: It's a common practice in Drupal to combine logic with html layouts while generating final output via functions, however. This approach makes Drupal designer to theme Drupal difficult, time consuming and most important limited to the knowledge of Drupal and programing

Solution: The Template API provides solution by separating the layout into template files while keeping logic in code and passing between via Template API interface. In addition, beside giving complete control and flexibility to the Designer, it also provides easy way to override template files, reuse logic while providing ability for anyone to customize Template API via hooks

Template API provides the following advantages:

  1. Quick, easy and simple process for designers to theme Drupal without knowledge of Drupal
  2. Create once and safe for future upgrades. With templates api, your themes are not effected by Drupal versions or upgrades. In another words, the same theme works for Drupal 7, Drupal 8 ...or Drupal 15
  3. Reuse Logic. (i.e. user_tapi, settings_tapi,etc)
  4. Possible Dynamic Templating - With Template Api you can swap different templates real time creating some really cool stuff!

Template API (TAPI)

Template API Overview

So there is template files and then there is Template API. The Template API is the interface points that developer makes available to the designer to insert in the template file. In simple words, the Template API is array of variables made available to the designer to insert in the template file. Furthermore, we have separated out the Template API into several smaller Template APIs such as breadcrumb_tapi, contact_tapi, etc each providing its specific functionality

In the following is a list of all Template API available with each serving its own responsibility described:

TAPI Name Interface Scope Description
Header TAPI $header_tapi (see details at 'Header TAPI' section) HTML Template API used in header for logo, site name, etc.
Footer TAPI $footer_tapi (see details at 'Footer TAPI' section) HTML Template API used in footer
Page TAPI $page_tapi (see details at 'Page TAPI' section) PAGE Template API for basic page info such as title, info, slogan, body
Breadcrumbs TAPI $breadcrumbs_tapi (see details at 'Breadcrumbs TAPI' section) breadcrumbs.tpl.php list of breadcrumbs in breadcrumbs template API format
Contact TAPI $contact_tapi (see details at 'Contact TAPI' section) contact.tpl.php all related info for contact form provided by Drupal core Contact module
Comment TAPI $comment_tapi (see details at 'Comment TAPI' section) contact.tpl.php all info for themeing comment form
Status Message TAPI $status_message_tapi (see details at 'Status Message TAPI' section) status-message.tpl.php info for 4 differnt messages - warning, info, danger and status
Menu TAPI $menu_tapi (see details at 'Menu & Link TAPI' section) main-menu.tpl.php, user-menu.tpl.php, bottom-menu.tpl.php this is the outer layout of the some menu that doesn't include the menu items themselves
Menu Item TAPI $link_tapi (see details at 'Menu & Link TAPI' section) main-menu-link.tpl.php, user-menu-link.tpl.php,bottom-menu-link.tpl.php this is single menu item itself part of menu
Search TAPI $search_tapi (see details at 'Search TAPI' section) search.tpl.php all info for search form
User TAPI $user_tapi (see details at 'User TAPI' section) HEAD and PAGE(if path with word 'user') The user TAPI includes registration, login and password reset info a long the user profile info
Registration TAPI $registration_tapi (see details at 'Registration TAPI' section) registration.tpl.php all info for registration form
Login TAPI $login_tapi (see details at 'Login TAPI' section) login.tpl.php all info for login form
Password Reset TAPI $password_reset_tapi (see details at 'Password Reset TAPI' section) password-reset.tpl.php all info for password reset form
Theme Settings TAPI $theme_tapi (see details at 'Theme Settings TAPI' section) Anywhere you include it typically in theme-settings.php The standard info for Theme Settings such as color, bg, sidebar position,etc

Note, the scope is the space each Template API interface points are available by default. The PAGE scope is the THEME_preprocess_page($var) function. The HTML scope is THEME_preprocess_html($var) function. All other scopes are within the specified template files. It's important to note that The Template API is not limited to only these scope. You can include any TAPI by calling its TAPI function(see the function name in each TAPI details section of your interest)

Triggers

The "DesignsSquare.com TEMPLATE API" is flexible enough to trigger Template API mechanism for one element but not for another. After enabling the module, in order for template API mechanism to start, make the template file present. For example, by placing the main-menu.tpl.php in the THEME/templates/template-api dir but not search.tpl.php file, the Template API mechanism will be applied to Main Menu element but not for Search Form. In another words, by having the template file present in THEME/templates/template-api, the template api will kick in for that element.

Furthermore, for two elements - Header and Footer there is an additional step. You have to include the Header or Footer elements in your template files themselves for Template API mechanism to work.

Here is quick overview of triggers for each element

Element Trigger Sample
Header
  1. template file - header.tpl.php needs to be present
  2. explicitly specify to import
 In the HTML scope(html.tpl.php), import as following:
<?php print $header ?>
Footer
  1. template file - footer.tpl.php needs to be present
  2. explicitly specify to import
 In the HTML scope(html.tpl.php), import as following:
<?php print $footer ?>
Breadcrumbs, Search, All Menus, Status Message, Login, Registration and Password Reset
  1. template file - ELEMENT.tpl.php needs to be present
 By placing template file into THEME/templates/template-api, starts the mechanism
Theme Settings you have to explicitly include as following:
module_load_include('inc', 'designssquare_lib_template_api', 'includes/template_api');
$theme_tapi = _theme_settings_tapi();

Menu Template API

Menu Templates

The module supports three menus - main-menu, bottom-menu and user-menu. Each of the menu has two different templates - NAME-menu.tpl.php and NAME-menu-link.tpl.php.

The template NAME-menu.tpl.php specifies the link wrapper the 'ul' tags layout. In another hand, the template NAME-menu-link.tpl.php specifies the link layout. Since menu can have sub menus, the template NAME-menu.tpl.php may be called several times. You can use the "depth" variable available to distinguish and theme accordingly

Menu & Link TAPI Description

There are two parts of Menu Template API - menu_tapi and link_tapi. The menu_tapi is available in teplate NAME-menu.tpl.php where is the link_tapi is in template NAME-menu-link.tpl.php. In the following is each description:

Content Type API Methods Implementing API Scope
MENUE TAPI $menu_tapi['depth']::[NUMBER](the depth of the menu. it can be sub-menu as well with different depth)
$menu_tapi['links']::[ARRAY](the links of the menu)
$menu_tapi['links'][N]['link_output']::[RENDERABLE ARRAY](the link of the menu to render via drupal_render)
_menu_tapi($link['below'], $depth) MENU-NAME-menu.tpl.php
LINK TAPI $link_tapi['title']::[STRING](menu link name)
$link_tapi['url']::[STRING](path to page)
$link_tapi['active']::[BOOLEAN](if link part of active trail)
$link_tapi['depth']::[NUMBER]
$link_tapi['has_sub_menu']::[BOOLEAN](if link has sub menu)
$link_tapi['sub_menu']::[RENDERABLE ARRAY](theme function to render sub menu)
_menu_link_tapi($link['link'], $depth, $link['sub_menu']) MENU-NAME-menu-link.tpl.php

Contact Template API

Contact Templates

For all of the pages, there is 3 different contact templates available:

  • contact.tpl.php
    This template usually used for main contact form in the contact us page or anywhere else part of the content section
  • contact-block.tpl.php
    This template usually used for sidebar, header, footer wheneever it is of form of block.
  • contact-inquiry.tpl.php
    This template usually is used for inquiry purpose like you have store and for each product there is an inquiry form

With multiple templates, you can render more than one contact form on the page at the same time. Each contact template recieves the same contact_tapi interface points. The only difference is that they have different contact form category, so when you receive email those are distinguished

To render any of the contact forms at any place in page scope:
print render($contact_form)
print render($contact_block_form)
print render($contact_inquiry_form)

Contact API utilizes Drupal core module - Contact, so it needs to be enabled in order for Contact Template API to work

Customize Template API & Templates

There may be time you like to customize the template API or templates files themselves. Whether is the case to add new field to registration form or alter menu link to uppercase letters, the Template API provides hooks to easy accomplish it

  1. To Alter Template API

    To Alter Template API use hook - hook_NAME-OF-TAPI_alter in your custom module. With this hook you able update any of the Template API interfaces. Here is a sample to modify main menu link title to upper case letters
    //alter main menu link
    function MODULE_menu_link_tapi_alter(&$link_tapi, $depth, $menu_name){
      if($menu_name == 'main_menu'){
       $link_tapi['title'] = strtoupper($link_tapi['title']);
      }
    }

    Here, we use hook to alter Menu Link TAPI. In the hook, we ensure we only alter for main menu links and not others by if condition. At last, we change the title and update the interface point 'title'

    Template API Hooks

    In the following are all the available hooks for altering Template API

    • hook_breadcrumb_tapi_alter(&$breadcrumb_tapi)
    • hook_theme_tapi_alter(&$theme_tapi)
    • hook_page_tapi_alter(&$page_tapi)
    • hook_header_tapi_alter(&$header_tapi)
    • hook_search_tapi_alter(&$search_tapi)
    • hook_contact_tapi_alter(&$contact_tapi)
    • hook_comment_tapi_alter(&$comment_tapi)
    • hook_user_tapi_alter(&$user_tapi)
    • hook_menu_tapi_alter(&$menu_tapi, $menu_name)
    • hook_menu_link_tapi_alter(&$link_tapi, $depth, $menu_name)
    • hook_registration_alter(&$registration_tapi)
    • hook_login_alter(&$login_tapi)
    • hook_password_reset_alter(&$password_reset_tapi)

    The name of hooks should be self explanatory of what TAPI it modifies as well as the parameter name

  2. To Alter Template File

    To Alter Template File it needs to be overriden via hook - hook_NAME-OF-TEMPLATE_template_alter in your custom module.
    1. Copy the template file you like to modify from themes/THEME/templates/template-api directory into your custom module directory
    2. Make the custom changes in the template file
    3. Tell to override old template file with your new one by specifying path via hook
      function MODULE_TEMPLATE-FILE-NAME_template_alter(&$template_path){
        $template_path = drupal_get_path('module','MODULE') . '/templates/template-api';
      }
       Here, the new custom template is stored in the custom module directory - 'sites/all/modules/MODULE-NAME/templates/template-api/TEMPLATE-FILE.tpl.php'. By specifying the new path, it will use the custom template file instead the default one provided by the theme
    4. Refresh cache

How To

How to Add New Field to Registration Form?

It may a case, that there is a need to add an additional field to the registration form. Since the registration form is in its own template file - registration.tpl.php, you have override the template file and alter Registration TAPI(optional) as following

  1. Create your custom module to keep your changes
  2. Update Template API via hook. The registration template file receives the array of variables(i.e. $registration_tapi) also referred as Registration TAPI(see "Registration TAPI" section) that we need to update as following:
    function MODULE_registration_tapi_alter(&$registration_tapi){
      $registration_tapi['address'] = array();
      $registration_tapi['address']['name'] = 'name';
      $registration_tapi['address']['type'] = 'text;
      $registration_tapi['address']['text'] = 'Address:';
    }
     Here we added, the name and type of the input field along the label text. This makes the address input field values available in the registration template file
  3. Override template with your new template.To override the current registration template, copy the existing one from sites/all/themes/THEME/templates/template-api/registration.tpl.php into your custom module directory. Lets update the template file with a new field:
    <form>
    ...
    <div class="form-group">
     <label ><?php print $registration_tapi["address"]["text"];?></label>
      <div class="input-icon">
       <i class="fa fa-check"></i>
       <input class="form-control placeholder-no-fix" type="<?php print $registration_tapi["address"]["type"];?>"
       placeholder="<?php print $registration_tapi["address"]["text"];?>"
       name="<?php print $registration_tapi["address"]["name"];?>" />
     </div>
    </div>
    ...
    </form>
     Here, the new input field is included with php printing our new registration TAPI values. At last, override the current path to the registration template with the path of our custom template file in our custom module as following:
    //override template file
    function MODULE_registration_template_alter(&$template_path){
      $template_path = drupal_get_path('module','ds_com_test_template_api') . '/path/to/template';
    }
     Here, we use the hook to replace the existing path to registration.tpl.php to our new one, so it uses our custom registration template file instead the default
  4. Refresh Cache
How to Pass Variables To Header?

Lets say you have user menu that needs to be displayed in the header. To pass user menu(in our case renderable array) to the header, there are two ways

  1. TAPI HOOK Each Tempalte API has hook including Header TAPI Via hook_header_tapi_alter(), you can pass the variable containing rendearable array
  2. Via Header Variable Template API usesd theme function that is later rendered. This theme function takes variable vars_tapi that you can update with user menu and access in the header scope
    MODULE_preporcess_html(){
     $vars['header']['#vars_tapi']['user_menu'] = array(
      '#theme' => 'template_api_user_menu',
      '#menu_tapi' => _menu_tapi($tree, $user_menu),
     );
    }
     Here, we attach renderable array for user meny and then we will render in the header.tpl.php as following: print render($vars_tapi['user_menu']);
